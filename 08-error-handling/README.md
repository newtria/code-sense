# 08. 에러 핸들링

## 이 폴더에서 배우는 것

- 에러 핸들링의 원칙 (실패는 반드시 일어난다)
- 커스텀 에러 클래스와 에러 계층 구조
- 일관된 에러 응답 형식
- 비동기 에러 처리 패턴
- 글로벌 에러 핸들러

## 실습

```bash
cd examples
node error-handling-demo.js  # 에러 핸들링 패턴 비교
```

## 핵심 질문

1. `catch (err) {}`처럼 빈 catch 블록이 왜 위험한가?
2. 에러 응답에 스택 트레이스를 노출하면 어떤 문제가 생기는가?
3. async 함수에서 try/catch 없이 await를 쓰면 무슨 일이 생기는가?
4. 모든 에러에 500을 반환하면 클라이언트는 어떤 어려움을 겪는가?
